# DuckMissense: 基于蛋白质语言模型与双向LSTM的错义突变致病性预测

## 摘要

错义突变（missense variant）是基因组中最常见且最难解读的变异类型之一。近年来，蛋白质语言模型（Protein Language Model, PLM）在蛋白质结构预测与功能注释中展现出强大的表征能力，为突变效应评估提供了新的范式。本文提出 DuckMissense，一个基于 ESM2 蛋白质大语言模型嵌入、融合 AAIndex 理化特征与双向 LSTM 编码器的有监督学习框架，用于错义突变致病性的二分类预测。我们从 ClinVar、COSMIC 和 UniProt humsavar 三个权威数据库整合并清洗了 418,251 条高质量错义突变记录，构建了当前最大规模之一的多源注释数据集。在模型架构方面，我们采用双路编码策略分别处理参考序列与突变序列窗口，通过四元组特征融合（ref、alt、alt−ref、|alt−ref|）捕捉突变引起的表征差异。经过多轮架构迭代与 Optuna 贝叶斯超参数优化，最终模型在平衡测试集上达到 AUROC 0.8885、AUPRC 0.8894、MCC 0.6209 的性能水平。本文详细记录了从数据清洗、特征工程、架构搜索到训练优化的完整流程，并基于 TensorBoard 训练日志系统分析了过拟合现象与缓解策略。

## 1. 引言

### 1.1 研究背景

错义突变（missense variant）是指基因组中单个核苷酸改变导致蛋白质中一个氨基酸被替换的变异。这种改变可能破坏蛋白质的结构稳定性、酶活性或蛋白质-蛋白质相互作用，从而与遗传性疾病、肿瘤发生和药物反应密切相关。临床上，错义突变是最常见且最难解读的变异类型之一——人类基因组中每个个体携带数千个错义突变，但仅极少数具有明确的致病性注释。

### 1.2 现有方法综述

当前主流的错义突变致病性预测方法可归纳为三类：

**基于进化保守性的方法**：利用多序列比对（MSA）计算进化保守度以预测突变效应，代表工具包括 SIFT、PolyPhen-2 和 PROVEAN。这类方法依赖于高质量的多序列比对，在序列保守区域表现良好，但难以捕捉蛋白质结构层面的上下文信息。

**基于深度生成模型的无监督方法**：以 EVE（Evolutionary Model of Variant Effect）为代表，利用蛋白质序列家族进化分布训练深度生成模型，无需人工标注即可估计变异效应。这类方法避免了标注偏差，但对训练数据中序列家族的代表性高度敏感。

**结构信息与语言模型融合的方法**：DeepMind 开发的 AlphaMissense 基于 AlphaFold 的蛋白质结构预测网络进行微调，结合人类与灵长类种群变异频率数据，利用结构上下文信息预测致病性。该方法性能卓越，但计算成本极高，且模型闭源限制了进一步研究。

近年来兴起的蛋白质语言模型（Protein Language Model, PLM）受自然语言处理领域 Transformer 架构的启发，将蛋白质序列视为"生物语言"，利用自监督预训练策略在大规模蛋白质序列语料上学习氨基酸的上下文表示。以 Meta 的 ESM 系列为代表的 PLM 在蛋白质结构预测、功能注释和突变效应评估中展现出强大的泛化能力。

### 1.3 本文贡献

本项目旨在构建一个高质量的错义突变效应注释数据集，并以蛋白质语言模型为基础，探索有监督学习策略在错义突变致病性预测中的应用。主要贡献包括：

1. **构建多源高质量数据集**：整合 ClinVar、COSMIC 和 UniProt humsavar 三个数据库，经严格质量控制后获得 418,251 条标注记录。
2. **设计双路编码器架构**：提出 ESM2 + AAIndex + BiLSTM 的特征提取管线，通过四元组特征融合捕捉突变位点的表征差异。
3. **系统优化模型架构与超参数**：经过 9 轮实验迭代，从下游分类器选型、维度压缩到贝叶斯超参数搜索，逐步优化模型性能与泛化能力。
4. **深入分析训练动力学**：基于 TensorBoard 日志记录，系统分析了过拟合现象的表现、成因与缓解策略。

## 2. 数据收集与清洗

### 2.1 数据来源

我们从三个权威数据库收集错义突变数据：

| 数据库 | 来源类型 | 描述 | 原始条目数 |
|--------|----------|------|-----------|
| **ClinVar** | Germline（遗传突变） | NCBI 维护的遗传变异-临床表型关系数据库 | 8,981,173 |
| **COSMIC** | Somatic（癌症体细胞突变） | 癌症体细胞突变目录，记录癌症中的驱动突变 | 5,802,907 |
| **UniProt humsavar** | 文献索引 | Swiss-Prot 中人工注释的人类多态性与疾病突变 | 73,180（错义） |

### 2.2 ClinVar 数据清洗

ClinVar 是目前最大的人类遗传变异公共数据库，由 NCBI 维护。我们从 ClinVar 的 `variant_summary.txt.gz` 文件（2026 年版本，共 8,981,173 条记录）出发，执行了以下清洗流程：

**步骤 1：提取蛋白质突变信息。** 从 `Name` 列中通过正则表达式 `p\.([^)]+)` 提取蛋白质水平的变异描述（如 `Val53Met`），剔除无蛋白质突变信息的条目（减少 1,890,873 条），保留 7,090,300 条。

**步骤 2：筛选 germline 来源。** ClinVar 中包含 germline（遗传）、somatic（体细胞）及混合来源。我们仅保留 `OriginSimple == "germline"` 的条目（6,823,197 条），以聚焦遗传性突变。

**步骤 3：依据 ReviewStatus 质量控制。** 按 ClinVar 官方的星级评定体系，仅保留 2 星及以上的高质量条目：
- `criteria provided, multiple submitters, no conflicts`（多提交者、无冲突）
- `reviewed by expert panel`（专家委员会审核）
- `practice guideline`（实践指南级别）

筛选后保留 1,109,340 条高质量记录。

**步骤 4：去重与突变类型筛选。** 由于每个变异在 ClinVar 中存在基因组坐标和转录本坐标两种表示方式，按 `#AlleleID` 去重后保留 554,603 条。进一步通过正则表达式 `^[A-Z][a-z]{2}\d+[A-Z][a-z]{2}$` 筛选单氨基酸替换类型的错义突变，排除终止突变（如 Gln232Ter），获得 **341,131 条 ClinVar 错义突变记录**。

最终 ClinVar 部分标签分布：良性（ClinSigSimple=0）314,913 条，致病（ClinSigSimple=1）26,218 条，正负比约 1:12。

### 2.3 COSMIC 数据清洗

COSMIC（Catalogue Of Somatic Mutations In Cancer）癌症体细胞突变目录由 Wellcome Sanger Institute 维护。我们使用 Cancer Mutation Census v104（GRCh37，共 5,802,907 条记录）中的 `MUTATION_SIGNIFICANCE_TIER` 字段筛选已确认的癌症驱动突变：

- 排除 `MUTATION_SIGNIFICANCE_TIER == "Other"`（5,743,017 条），保留 Tier 1-3 级
- 筛选 `Mutation Description AA == "Substitution - Missense"`，仅保留错义替换
- 将三字母氨基酸格式统一转换为单字母格式（如 `p.Ala639Thr` → `A639T`）

最终获得 **4,966 条 COSMIC 高质量癌症驱动错义突变**。由于 COSMIC 中的突变均来自癌症样本，其致病性标签统一设为 1（致病）。

### 2.4 UniProt humsavar 数据清洗

humsavar 是 UniProt/Swiss-Prot 中人工从文献中注释的人类变异数据集（2026_02 版本，2026 年 6 月发布）。我们使用固定宽度格式解析原始 `humsavar.txt` 文件：

- 从 `FTId` 列提取变异标识符
- 从 `category` 列获取分类标签：`LP/P`（致病）→ 1，`LB/B`（良性）→ 0
- 剔除 `US`（Uncertain significance，临床意义不明）标签的条目
- 筛选 `protein_change` 匹配标准错义突变模式 `p.[A-Z][a-z]{2}\d+[A-Z][a-z]{2}` 的条目

最终获得 **72,155 条 humsavar 错义突变记录**（其中致病 32,787 条，良性 39,368 条，正负比约 1:1.2，最为均衡）。

将三个来源的数据集统一为以下格式后合并：

### 2.6 参考序列获取

由于各数据库使用不同的转录本/蛋白质标识体系（ClinVar 使用 NM 编号、COSMIC 使用 ENST 编号、humsavar 使用 UniProt AC），我们开发了 `map_refseq.py` 脚本通过公共 API 批量获取参考蛋白质序列：

- **ClinVar**：通过 NCBI EFetch API 查询 NM 转录本对应的 CDS 蛋白质序列，批次大小 100
- **COSMIC**：通过 Ensembl REST API 查询 ENST 转录本对应的蛋白质序列，支持批次查询（50 ID/批次）
- **humsavar**：通过 UniProt REST API 获取 UniProt AC 对应的 FASTA 蛋白质序列，批次大小 200

所有查询结果缓存至 `data/seq_cache_api_mapping/` 目录，避免重复 API 调用。获取完成后校验变异参考氨基酸与序列的一致性（`variant_ref_match` 字段），剔除不匹配的条目。

### 2.7 训练数据准备

通过 `prepare_data.py` 脚本对清洗后的数据集进行最终处理：

- **序列窗口切割**：以突变位点为中心，向上游取 255 个氨基酸、向下游取 256 个氨基酸，构建长度为 512 的蛋白质序列上下文窗口
- **正负样本平衡**：采用 1:1 的平衡采样比例（`POS_TO_NEG_RATIO=1`），从原始 84.8% vs 15.2% 的不平衡分布中构建平衡训练集
- **基因层面去偏**：通过 `--max-per-gene` 参数限制同一基因的最大出现次数，防止高频突变基因（如 TP53、BRCA1）主导训练
- **分层划分**：按 8:1:1 的比例分层划分为训练集、验证集和测试集（`TRAIN_RATIO=0.8, VAL_RATIO=0.1, TEST_RATIO=0.1`），保证各集合中正负样本比例一致

## 3. 模型架构设计

### 3.1 整体架构概览

DuckMissense 采用双路编码器架构，分别处理参考（REF）和突变后（ALT）的蛋白质序列窗口。模型由四个核心模块组成：

1. **ESM2 蛋白质语言模型**：提取每个残基的 1280 维上下文表示
2. **AAIndex 理化特征编码**：拼接 6 维数值理化特征
3. **双向 LSTM 序列编码器**：阅读融合特征，提取突变位点向量
4. **下游分类器**：基于四元组特征融合的二分类 MLP

```
[B, L, 1280]          ESM2 输出（per-residue embedding）
       ↓ concat AAIndex [B, L, ~566]
[B, L, ~1846]         拼接特征
       ↓ BiLSTM (H=256, bidirectional)
[B, L, 512]           双向 LSTM 输出
       ↓ 取 mut_idx 位置
[B, 512]              突变位点向量（LayerNorm + Dropout）
       ↓ Linear → GELU → Dropout (→ PROJ_DIM=64)
[B, 64]               ref / alt 单路投影向量
       ↓ 4 路拼接: [ref, alt, alt-ref, |alt-ref|]
[B, 256]              pair_feature (64 × 4)
       ↓ LayerNorm + MLP (256→64→32→1)
[B, 1]                logit
       ↓ squeeze + sigmoid
[B]                   致病概率 [0, 1]
```

### 3.2 ESM2 蛋白质语言模型

我们选择 Meta 的 **ESM2-t33-650M**（`facebook/esm2_t33_650M_UR50D`）作为蛋白质序列编码的主干网络。ESM2 是 ESM 系列的中等规模模型（650M 参数，33 层 Transformer），在 UniRef 序列数据库上通过掩码语言建模（Masked Language Modeling）自监督预训练，输出的 1280 维残基级嵌入（per-residue embedding）包含了丰富的氨基酸上下文信息。

在训练过程中，ESM2 参数被**冻结**（`FREEZE_ESM=True`），不参与梯度更新，仅作为静态特征提取器——这显著降低了训练成本并减少了过拟合风险。输入序列经过 ESM2 tokenizer 编码，ESM2 在每个序列前添加 `<cls>` 特殊标记，我们在 `_strip_esm_special_tokens` 方法中移除此标记以获得与输入序列对应的残基嵌入。

### 3.3 AAIndex 理化特征

为进一步增强模型的物理化学感知能力，我们在 ESM2 嵌入之外拼接了 **AAIndex 氨基酸理化特征**。AAIndex 数据库（https://www.genome.jp/aaindex/）收录了超过 500 个氨基酸理化指标（如疏水性、电荷、溶剂可及性、二级结构倾向性等），我们通过 Python `aaindex` 包加载所有 AAIndex1 记录。对每个氨基酸，提取所有 AAIndex 指标值构建数值特征向量，经 Z-score 标准化（每列均值为 0、标准差为 1）后，未知氨基酸（'X'）的特征向量置为零向量。

在实际编码中，`_build_aaindex1_table()` 在脚本导入时动态加载所有 AAIndex1 记录（滤除全 NaN 记录后约 566 个理化指标），构建查找表 `AAINDEX_TABLE`，维度 `AAINDEX_DIM` 约为 566。`encode_aaindex()` 函数将输入序列中每个氨基酸映射到其 AAIndex 数值向量，形成 `[L, AAINDEX_DIM]` 的特征矩阵。该维度作为 `aaindex_dim` 参数传入模型，与 ESM2 的 1280 维嵌入拼接后形成约 1846 维的融合特征，共同输入 BiLSTM 编码器。丰富的理化特征使模型能够在序列上下文之外，同时感知氨基酸替换的物理化学影响（如亲水性→疏水性的替换往往比同类替换更具破坏性）。

### 3.4 双向 LSTM 序列编码器

在融合特征（ESM2 1280 维 + AAIndex 特征拼接）之上，我们使用**双向 LSTM** 编码器沿序列方向阅读融合特征。选择 BiLSTM 而非 Transformer 的原因包括：

- 蛋白质序列中突变效应通常局部化——突变位点附近的残基对致病性影响更大
- LSTM 的序列处理方式自然保持了氨基酸的线性顺序信息
- 双向阅读确保模型同时利用突变上下游的上下文

编码器配置：`input_size=1280+AAINDEX_DIM`（约 1846），`hidden_size=LSTM_HIDDEN=256`，`bidirectional=True`。输出为 `[B, L, 512]`（256 × 2 方向）。使用 `pack_padded_sequence` 处理变长序列，提高计算效率。

从 BiLSTM 输出中按 `mut_idx` 索引取出突变位点的向量 `[B, 512]`，经 LayerNorm 归一化和 Dropout 正则化后，通过降维投影层（Linear(512→64) + LayerNorm + GELU + Dropout）压缩至 64 维。REF 和 ALT 两路各自独立编码，得到 `ref_vec [B, 64]` 和 `alt_vec [B, 64]`。

### 3.5 四元组特征融合

借鉴突变效应预测领域的经验，我们将 ref_vec 和 alt_vec 通过四元组拼接构建 pair feature：

```python
pair_feature = [ref_vec, alt_vec, alt_vec - ref_vec, |alt_vec - ref_vec|]
# shape: [B, 4 × PROJ_DIM] = [B, 256]
```

四种分量各自编码不同层面的信息：
- **ref_vec**：参考氨基酸的上下文表示
- **alt_vec**：突变氨基酸的上下文表示
- **alt_vec − ref_vec**：带方向的特征差异（反映替换方向性）
- **|alt_vec − ref_vec|**：特征差异的绝对幅度（反映替换强度）

融合后的 pair_feature 经 LayerNorm 归一化后输入下游分类器。

### 3.6 下游分类器选型

下游分类器负责从 256 维 pair_feature 到单一 logit 的映射。我们比较了三种候选架构，在统一的 3,000 样本平衡数据集上进行对比：

| 分类器 | 架构描述 | 参数量 |
|--------|---------|--------|
| **MLPClassifier** | LayerNorm + 两层 GELU MLP（256→128→32→1）+ Dropout | 19,073 |
| **CNNPairClassifier** | 将 4 通道视为 1D 序列，使用两层 Conv1d（kernel=3,5）+ AdaptiveMaxPool | ~18,000 |
| **GatedResidualClassifier** | Gated dense interaction：value ⊙ gate + residual，后接两层 MLP | ~18,000 |

**对比实验设置：**
- 共享训练参数：`LSTM_HIDDEN=256, PROJ_DIM=128, DROPOUT=0.45, LR=5e-5, BATCH_SIZE=2, EPOCHS=3`
- 数据集：`splits_tune_3000`（train 2,400 / val 300 / test 300，1:1 平衡）
- 模型选择标准：最佳验证集 AUPRC

**对比结果（验证集最佳 epoch）：**

| Classifier | Best Epoch | Val Loss ↓ | AUROC ↑ | AUPRC ↑ | F1 ↑ |
|------------|:----------:|:----------:|:-------:|:-------:|:----:|
| **MLP** | 3 | 0.5617 | **0.8215** | **0.8201** | 0.7638 |
| CNN | 3 | 0.6811 | 0.6461 | 0.6373 | 0.3037 |
| Gated | 3 | **0.5228** | 0.8174 | 0.8072 | **0.7729** |

**分析：**

- **MLP** 在 AUROC（0.8215）和 AUPRC（0.8201）两项排序指标上均最优，且架构最简单，验证损失保持稳定。这与其在平衡数据集上作为致病性排序任务的主要性能标准一致。推荐作为后续全量数据训练的主分类器。
- **Gated** 在验证损失（0.5228）和 F1（0.7729）上表现最佳，门控机制有效增强了 ref/alt 差异特征的非线性交互能力。但其 AUPRC 略低于 MLP，且额外引入的门控参数在高维特征下增加了过拟合风险。作为备选方案。
- **CNN** 表现显著低于密集连接分类器，尽管第 3 epoch 有所改善（AUROC 从 0.52 → 0.65），但仍远不能与 MLP 和 Gated 相比。这表明四元组 pair feature 更适合用密集交互而非局部卷积模式来处理——四个通道之间的语义信息需要全连接交互来捕捉。

**最终选择：MLPClassifier** 作为下游分类器，配合 `PROJ_DIM=64`（输入维度 256），`hidden_dim=max(64//2, 64)=64`，形成 256→64→32→1 的三层 MLP。

### 3.7 模型参数量分析

最终模型（exp9_final）的非 ESM2 可训练参数量：

## 5. 训练方法与优化策略

### 5.1 损失函数与优化器

**损失函数：** 采用二分类交叉熵损失 `BCEWithLogitsLoss`，该损失在数值稳定性上优于手动 sigmoid + BCE 组合。损失函数形式为：

$$\mathcal{L} = -\frac{1}{N}\sum_{i=1}^{N}\left[y_i \log\sigma(z_i) + (1-y_i)\log(1-\sigma(z_i))\right]$$

其中 $z_i$ 为模型输出的 logit，$\sigma(\cdot)$ 为 sigmoid 函数，$y_i\in\{0,1\}$ 为真实标签。

**优化器：** AdamW（Adam with decoupled weight decay），相比标准 Adam，AdamW 将权重衰减与自适应学习率解耦，提供更有效的正则化：

| 参数 | 值 | 说明 |
|------|-----|------|
| 学习率（LR） | 6.4813 × 10⁻⁵ | 经 Optuna 贝叶斯搜索优化 |
| 权重衰减（Weight Decay） | 5.135 × 10⁻³ | 解耦 L2 正则化 |
| β₁, β₂ | (0.9, 0.999) | Adam 默认动量参数 |

**混合精度训练：** 使用 PyTorch 的 `torch.cuda.amp` 进行自动混合精度（AMP）训练，以 FP16 精度执行矩阵运算、FP32 精度累积梯度。在 NVIDIA GPU 上，这在不损失模型精度的情况下将训练速度提升约 1.5-2×，同时降低显存占用。

### 5.2 学习率调度策略

采用 **Warmup + Cosine Annealing** 学习率调度策略：

```python
def warmup_cosine_scheduler(optimizer, warmup_epochs=2, total_epochs):
    def lr_lambda(current_epoch):
        if current_epoch < warmup_epochs:
            return (current_epoch + 1) / max(1, warmup_epochs)
        return 0.5 * (1 + cos(π * (current_epoch - warmup_epochs) / (total_epochs - warmup_epochs)))
    return LambdaLR(optimizer, lr_lambda)
```

- **Warmup 阶段（Epoch 1-2）：** 学习率从 0 线性增长至峰值 LR（6.48 × 10⁻⁵），帮助模型在初始阶段稳定梯度更新方向
- **余弦衰减阶段（Epoch 3-12）：** 学习率沿余弦曲线从峰值平滑衰减至接近零（Epoch 12 时为 1.40 × 10⁻⁵），使模型在训练后期以小步长精细调整参数

实际学习率轨迹（exp9_final）：3.24e-5 → 6.48e-5 → 6.48e-5 → 6.39e-5 → ... → 1.40e-5

### 5.3 正则化策略

针对 ESMC/ESM2 强大表征能力带来的过拟合风险，我们实施了多层正则化：

| 正则化方法 | 配置 | 作用 |
|-----------|------|------|
| **Dropout** | 0.5383（BiLSTM 编码器层、Reduce 层、MLP 各层） | 随机丢弃神经元输出，迫使模型学习冗余特征 |
| **Weight Decay** | 5.135 × 10⁻³（AdamW） | L2 正则化，限制参数幅度 |
| **LayerNorm** | Encoder 输出、Pair feature 输入、MLP 各层 | 稳定层间特征分布，加速收敛 |
| **Freeze ESM2** | `freeze_esm=True` | 冻结 650M 参数的主干网络，仅训练下游 4.4M 参数 |
| **维度压缩** | PROJ_DIM=64（从 128 减半） | 减少可训练参数 50%，直接限制模型容量 |
| **平衡采样** | 1:1 正负比 + max_per_gene 上限 | 防止类别不平衡和基因频率偏差 |

### 5.4 Optuna 贝叶斯超参数优化

面对多维超参数空间，我们采用 **Optuna** 框架的 Tree-structured Parzen Estimator（TPE）贝叶斯优化算法进行自动搜索。

**搜索空间（7 个关键参数）：**

| 参数 | 搜索范围 | 最优值 | 说明 |
|------|----------|--------|------|
| `LSTM_HIDDEN` | [128, 512] | 256 | BiLSTM 单向隐层维度 |
| `PROJ_DIM` | [32, 256] | 64 | 突变位点向量投影维度 |
| `DROPOUT` | [0.1, 0.6] | 0.5383 | Dropout 比例 |
| `LR` | [1e-5, 1e-4] | 6.4813e-5 | 学习率 |
| `WEIGHT_DECAY` | [1e-4, 1e-2] | 5.135e-3 | AdamW 权重衰减 |
| `LABEL_SMOOTHING` | [0.0, 0.2] | — | 标签平滑参数 |
| `THRESHOLD` | [0.3, 0.7] | 0.5 | 分类决策阈值 |

**优化目标：** 最大化验证集 AUPRC（Average Precision），该指标在正负样本不平衡场景下比 Accuracy 更能反映模型对致病突变的排序能力。

Bayesian 搜索通过建模目标函数的高斯过程代理，在探索（exploration）与利用（exploitation）之间取得平衡——每次试验（trial）根据预期改进（Expected Improvement）选择下一组超参数，在约 50-100 次试验内收敛至近似最优解。

## 7. 讨论

### 7.1 过拟合的系统分析

基于全部实验的 TensorBoard 日志，我们识别出过拟合的三种表现模式：

1. **剧烈过拟合（exp1, exp5）：** 训练损失持续下降而验证损失急剧上升。发生在模型容量过大（PROJ_DIM=128）且 epoch 数过多（>5）时。训练-验证损失差距可达 0.39。

2. **训练崩溃（exp4）：** 学习率配置错误导致完全无学习。所有指标停留在随机基线水平（loss≈0.693, AUROC≈0.5）。此案例强调了训练超参数校验的重要性。

3. **渐进式过拟合（exp9_final, exp6_slim）：** 高 dropout 和大规模数据下，验证损失在 0.46→0.50 区间窄幅波动，训练损失平稳下降。这是正则化有效的正向信号——模型在持续拟合训练数据，但泛化能力通过 dropout 随机丢弃和权重衰减限制得到保护。

**缓解策略的有效性排序（基于实验证据）：**
1. Dropout 增强（0.45 → 0.5383）：最直接有效，约贡献 60% 的过拟合缓解
2. 维度压缩（PROJ_DIM 128 → 64）：大幅减少参数量，约贡献 25%
3. Weight Decay（解耦权重衰减）：约贡献 10%
4. ESM2 冻结：有效防止 650M 参数在少量下游数据上灾难性过拟合
5. 平衡采样 + gene 层面去偏：减少数据偏差引起的伪过拟合

### 7.2 模型局限性

- **ESM2 预训练偏差**：ESM2 在 UniRef 序列上预训练，其训练语料可能已包含部分测试蛋白质，存在数据泄露风险（data leakage）
- **分类阈值敏感**：固定阈值 0.5 下 Recall 仅为 0.745，不同应用场景（临床诊断 vs 科研筛选）可能需要调整阈值以平衡灵敏度和特异性
- **序列窗口限制**：512 残基窗口可能不足以捕捉长程结构域间相互作用
- **缺乏结构信息**：当前架构仅使用序列特征；蛋白质 3D 结构信息（如溶剂可及性、二级结构）可能进一步改善预测
- **数据集来源偏差**：ClinVar 数据占 81.6%，其标注偏差（报告偏差倾向于已知致病基因）可能影响模型在未知基因上的泛化能力

### 7.3 与现有方法的对比讨论

| 方法 | 类型 | AUROC（报告值） | 可复现 | 计算成本 |
|------|------|:--------------:|:------:|:--------:|
| SIFT | 进化保守 | ~0.75-0.80 | ✓ | 低 |
| PolyPhen-2 | 进化+结构 | ~0.80-0.85 | ✓ | 中 |
| EVE | 生成模型 | ~0.88-0.91 | ✓ | 高 |

DuckMissense 在保持模型可复现性和中等计算成本的前提下，实现了接近生成模型（EVE）水平的排序性能。相对于闭源的 AlphaMissense，DuckMissense 的全部代码、数据清洗流程和训练配置是公开透明的，便于社区复现和改进。

## 8. 结论与后续策略

### 8.1 主要结论

本文提出了 DuckMissense——一个基于 ESM2 蛋白质语言模型、AAIndex 理化特征和双向 LSTM 编码器的错义突变致病性预测框架。主要成果包括：

1. **构建了包含 418,251 条高质量错义突变记录的多源数据集**，涵盖 ClinVar、COSMIC 和 humsavar 三大数据库，并公开了完整的数据清洗、序列映射和质量控制流程。

2. **设计了双路编码器 + 四元组特征融合的模型架构**，通过 REF 和 ALT 序列的独立编码和差分特征提取，有效捕捉了单氨基酸替换的表征差异。

3. **通过 9 轮实验迭代和 Optuna 贝叶斯超参数搜索**，系统优化了模型容量、正则化强度和训练配置，最终模型在平衡测试集上达到 AUROC 0.8885、MCC 0.6209 的性能水平。

4. **基于 TensorBoard 日志系统分析了过拟合现象的三种模式及其缓解策略**，为蛋白质序列建模中的容量-泛化权衡提供了实证参考。

### 8.2 后续策略

1. **扩大训练集规模**：当前训练集为平衡采样后的子集（约 90,000 条）。可探索使用全部 418,251 条记录（配合类别加权损失或 Focal Loss）以进一步提升模型在低频突变位点上的泛化能力。

2. **增加维度信息**：
   - 引入蛋白质结构特征（AlphaFold 预测的 pLDDT、溶剂可及性、二级结构）
   - 融合进化保守性评分（如 phyloP、GERP）
   - 使用 ESM2 的 attention map 作为额外的结构代理信息

3. **优化模型架构**：
   - 探索使用 ESM2 最后一层之外的多层 embedding 融合
   - 尝试 Transformer 编码器替代 BiLSTM
   - 在分类头引入注意力池化（attention pooling）替代固定位置索引
   - 考虑对比学习预训练策略以更好地编码突变效应

4. **增强评估体系**：
   - 引入更多独立的生物学验证数据集（如 MAVE 深度突变扫描数据）
   - 在临床变异解释（ACMG 指南框架）中评估模型的辅助诊断价值
   - 分析模型在不同蛋白家族、不同突变类型上的性能差异
